# Документація з розгортання сервісу нотаток

## Базовий образ та ОС
Для роботи сервісу необхідно використовувати офіційний образ дистрибутива **Debian 13 (Trixie)**.
* **Де взяти:** Офіційні дзеркала Debian.
* **Який завантажувати:** `debian-13-generic-amd64` (для віртуальних машин) або інсталяційний ISO `netinst` для архітектури x86_64.

## Вимоги до ресурсів ВМ
Для стабільної роботи компонентів (MariaDB, Nginx, Go App) рекомендуються наступні мінімальні характеристики:
* **CPU:** 1 Core (vCPU).
* **RAM:** 1 GB (мінімум), 2 GB (рекомендовано для комфортної збірки Go).
* **Disk:** 10 GB вільного простору.
* **Мережа:** Відкриті порти 80 (HTTP) та 22 (SSH).

## Налаштування ОС
* **Розбивка диска:** Стандартна (автоматична), вся система на одному розділі (`/`).
* **Користувач:** Під час інсталяції створіть користувача з правами `sudo`.

## Доступ до ВМ
* **Метод входу:** SSH або консоль віртуалізації.
* **Стандартні облікові дані (створюються скриптом):**
    * `student` / `12345678` (потребує зміни пароля при першому вході).
    * `teacher` / `12345678` (потребує зміни пароля при першому вході).
    * `operator` / `12345678` (обмежені права sudo на керування сервісом).

## Запуск розгортання
Перед запуском переконайтеся, що файл `.env` заповнений коректними даними для бази даних.
1. Завантажте файли проєкту на ВМ.
2. Перейдіть у директорію з проєктом.
3. Надайте скрипту права на виконання:
   ```bash
   chmod +x deploy.sh
   ```
4. Запустіть розгортання від імені суперкористувача:
   ```bash
   sudo ./deploy.sh
   ```

# Інструкція з тестування розгорнутої системи

Після завершення роботи скрипта необхідно провести перевірку коректності налаштувань за наступними кроками:

## Перевірка системних сервісів
Переконайтеся, що всі необхідні демони запущені та працюють:
```bash
systemctl status mariadb
systemctl status nginx
systemctl status mywebapp.service
```
*Очікуваний результат: Всі сервіси мають статус `active (running)`.*

## Перевірка доступності веб-інтерфейсу
Виконайте запит до сервера за допомогою `curl` або відкрийте IP-адресу ВМ у браузері:
```bash
curl -I http://localhost/healthy/alive
```
*Очікуваний результат: HTTP-статус `200 OK`.*

## Перевірка бази даних
Перевірте, чи створена база даних та користувач (використовуйте дані з `.env`):
```bash
mariadb -u <USER> -p -e "SHOW DATABASES;"
```
*Очікуваний результат: У списку присутня база даних, вказана в змінній `MARIADB_DATABASE`.*

## Перевірка користувачів та прав
Спробуйте увійти під користувачем `operator` та перезавантажити сервіс:
```bash
su - operator
sudo systemctl restart mywebapp.service
```
*Очікуваний результат: Сервіс перезавантажується без запиту пароля (згідно з налаштуваннями sudoers).*

## Перевірка файлової системи та лімітів
Перевірте наявність "залікової книжки" студента:
```bash
ls -l /home/student/gradebook
```
*Очікуваний результат: Файл існує, власник — `student`, права доступу — `644`.*
