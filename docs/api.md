# Ендпоінти

Цей API підтримує Content Negotiation для більшості ендпоінтів. Залежно від заголовка `Accept` у запиті, сервер може повертати дані у форматі `JSON` (`application/json`) або `HTML` (`text/html`).

## `GET /` (Index)
Повертає головну сторінку додатка.
**Відповідь:**
    - `200 OK`: Віддає шаблон `index.html`.

## `GET /notes`
Отримує список усіх існуючих нотаток.
**Відповідь:**
    - `200 OK`: Масив об'єктів нотаток (формат JSON або HTML-шаблон `notes.tmpl`).
    - `500 Internal Server Error`: У разі внутрішньої помилки сервера.

## `GET /note/{id}`
Отримує конкретну нотатку за її унікальним ідентифікатором.
**Параметри шляху:**
    - `id` (string) — унікальний ідентифікатор нотатки.
**Відповідь:**
    - `200 OK`: Об'єкт нотатки (формат JSON або HTML-шаблон `note.tmpl`).
    - `404 Not Found`: Якщо нотатку з таким `id` не знайдено.
    - `500 Internal Server Error`: У разі іншої внутрішньої помилки.

## `POST /notes`
Створює нову нотатку.
**Тіло запиту (Request Body):** JSON
    ```json
    {
      "title": "Назва нотатки",
      "content": "Текст нотатки"
    }
    ```
**Відповідь:**
    - `200 OK`: Успішне створення. Повертає `{"status": "success"}` у JSON або HTML-шаблон `status.html`.
    - `422 Unprocessable Entity`: Некоректне тіло запиту (наприклад, відсутні обов'язкові поля).
    - `500 Internal Server Error`: Помилка збереження бази даних/сервісу.

## `GET /healthy/alive`
Базова перевірка, чи запущений та чи відповідає сервіс.
**Відповідь:**
    - `200 OK`: Повертає рядок `OK`.

## `GET /healthy/ready`
Перевірка готовності сервісу приймати та обробляти запити (наприклад, чи встановлено з'єднання з базою даних).
**Відповідь:**
    - `200 OK`: Сервіс готовий, повертає рядок `OK`.
    - `500 Internal Server Error`: Сервіс не готовий до роботи (внутрішні компоненти недоступні).
